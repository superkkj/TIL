### [기준점 하나가 계층을 만든다] 루트 (Root)

> 🔑 핵심 키워드: `root` · `root reference` · `depth 0` · `subtree root` · `rotation` · `forest` [출처: 01-21-root.md §1]

웨이드님~ tree 용어 시리즈의 첫 주자, root야! 😊 root는 "그림의 맨 위 동그라미"가 아니라 계층 방향을 결정하는 기준점이라는 것, 그리고 root 참조 하나를 잘못 관리하면 tree 전체를 잃는다는 것까지 확인해 보자. 🤓

***

### 🧭 핵심 원리 (Core Principles & Concepts)

정석 정의부터. root는 **트리에서 시작점이 되는 노드**야. 일반적으로 부모가 없는 최상위 노드지. [출처: 01-21-root.md §1, NIST DADS Root (원본 §0 재인용)]

쉽게 말하면 조직도에서 대표, 파일 시스템에서 최상위 폴더 같은 위치야. 비유는 이해용이고 면접에서는 정석 용어로 돌아와 답해. [출처: 01-21-root.md §1]

root가 왜 필요하냐면, 트리 순회와 탐색은 보통 root에서 시작하기 때문이야. root가 있어야 아래 subtree들의 기준점이 생겨. [출처: 01-21-root.md §2]

이 문서의 범위는 root의 정의와 상위 자료구조에서의 역할까지야. 정렬·동시성·영속성·특정 시간 복잡도는 별도 불변식이나 구현 계약이 있을 때만 보장되고, 이름만 보고 성질을 단정하면 안 돼. [출처: 01-21-root.md §3]

#### 📖 어려운 말 먼저 풀기 (원본 용어 표 보존)

| 용어 | 쉬운 뜻 |
|---|---|
| root | 부모가 없는 트리의 시작 Node다. |
| root reference | Tree 객체가 시작 Node를 찾아갈 수 있게 보관하는 참조 값이다. |
| depth 0 | root에서 root까지 지나가는 edge가 0개라는 뜻이다. |
| subtree root | 전체 트리의 중간 Node를 그 아래 부분 트리의 시작점으로 볼 때의 root다. |
| rotation | BST의 정렬 순서는 유지하면서 부모·자식 연결을 바꿔 높이를 조정하는 연산이다. 전체 root에서 일어나면 root 참조도 바꿔야 한다. |

[출처: 01-21-root.md §1 어려운 말 먼저 풀기]

### 🏗️ 구조에서의 역할과 연산 (Role & Operations)

#### 🔗 root는 구조에서 어떤 역할을 하나

rooted tree에서 root는 모든 node로 가는 경로의 출발점이며 parent가 없는 유일한 node야. 저장 구조에서는 tree 객체가 root 참조 하나를 보관하고, 빈 tree라면 root를 null로 표현할 수 있어. [출처: 01-21-root.md §4]

```java
final class Tree<T> {
    Node<T> root; // null이면 empty tree
}
```

#### 🔀 root를 바꾸면 무엇이 바뀌나

같은 무방향 tree에서도 다른 node를 root로 선택할 수 있어. edge 자체는 같지만 parent, child, depth, subtree 범위가 바뀌어. 그래서 root는 단순히 "그림의 가장 위"가 아니라 계층 방향을 결정하는 기준이야. 원본 예시로 확인해 보자. [출처: 01-21-root.md §4]

```text
무방향 edge: A - B - C

A를 root로 선택       B를 root로 선택
A depth 0             B depth 0
└ B depth 1          ├ A depth 1
  └ C depth 2        └ C depth 1
```

edge 집합은 같지만 parent, child, depth, subtree, traversal 시작점이 달라져. [출처: 01-21-root.md §4]

> 😒 **Bailey**: 흥. "root는 그냥 맨 위에 그리는 노드잖아요"라고 하면 반쪽짜리 답이야. 같은 edge 집합에서 root 선택만 바꿔도 depth가 전부 달라진다는 걸 위 예시로 바로 보여줄 수 있어야지. [출처: 01-21-root.md §4]

#### ⚙️ root가 연산에서 사용되는 지점

연결 개념과 사용 지점을 원본 그대로 정리할게. [출처: 01-21-root.md §5]

```text
root
 ├─ child
 └─ child
```

- 전체 탐색: root에서 DFS/BFS를 시작한다.
- BST 검색: target 비교를 root에서 시작해 한 subtree를 선택한다.
- Heap: root가 최소값 또는 최대값이다.
- 균형 회전: root가 바뀌면 Tree 객체의 root 참조도 갱신해야 한다.
- 삭제: root가 유일한 node면 삭제 후 empty tree가 된다.

**root 교체 실패 사례**도 그대로 보존할게. 링크 회전은 했지만 `Tree.root`를 여전히 G로 두면 새 root P에서 시작할 수 없어 구조 일부에 접근하지 못하는 조용한 버그가 생겨. [출처: 01-21-root.md §5]

```text
회전 전 Tree.root = G

    G                 P
   /                 / \
  P       ->        X   G
 /
X
```

상태 추적은 위 연산 예시를 입력부터 결과까지 따라가며 값·index·Node·Stack·Queue 중 실제로 변하는 상태를 확인하는 방식으로 해. [출처: 01-21-root.md §6]

### 🧷 불변식과 경계 사례 (Invariants & Edge Cases)

불변식은 삽입·조회·삭제 전후에도 계속 참이어야 하는 규칙이야. 자료구조가 망가지지 않았다고 판단하는 필수 조건이고, 연산이 끝난 뒤에도 다시 맞아야 해. root 관련 구체 조건은 위 내부 구조와 연산 설명을 기준으로 확인하고, 상위 자료구조의 불변식과 충돌하는 변경은 허용하지 않아. [출처: 01-21-root.md §7]

경계 사례 세 가지는 꼭 기억해. [출처: 01-21-root.md §8]

- node 하나인 tree에서는 그 node가 root이면서 leaf다.
- 빈 tree의 root는 null로 표현할 수 있으며 height 규약은 자료마다 다르다.
- root가 여러 개라면 하나의 rooted tree가 아니라 여러 tree의 forest로 본다.

### 📊 성능과 선택 기준 (Cost & Selection)

root 자체가 독립 연산을 모두 정의하는 것은 아니야. 접근 비용은 배열·Node·Tree 등 실제 저장 표현과 상위 자료구조의 연산 계약을 기준으로 판단해. [출처: 01-21-root.md §9] 자료구조 선택도 이 개념만 떼어 하지 않고, 필요한 조회·삽입·삭제·정렬·범위·순회 연산 기준으로 비교해. [출처: 01-21-root.md §10] 근거는 표준·공식 자료 기준, Java API는 Oracle Java 17 문서 명시 범위에서만 보장으로 말하고, 실무에서는 구현체 API 계약·입력 크기·데이터 분포·변경 빈도·동시 접근 조건을 함께 확인해. [출처: 01-21-root.md §11, §12]

#### Q1. root가 여러 개인 구조도 트리인가?

<details><summary>답 보기</summary>

root가 여러 개인 구조는 보통 forest라고 부른다. 여러 트리의 집합으로 볼 수 있다. [출처: 01-21-root.md §14 Q1]

</details>

#### Q2. root를 찾는 비용은 항상 O(1)인가?

<details><summary>답 보기</summary>

Tree 객체가 root 참조를 보관하면 O(1)이다. 임의 node에서 parent만 따라 root를 찾는다면 node depth만큼 O(depth)가 든다. 표현 방식의 전제를 먼저 말해야 한다. [출처: 01-21-root.md §14 Q2]

</details>

#### Q3. root는 항상 값이 가장 작은 node인가?

<details><summary>답 보기</summary>

아니다. 일반 tree와 BST의 root는 최소값일 필요가 없다. min-heap은 추가 heap 불변식 때문에 root가 최소값이다. [출처: 01-21-root.md §14 Q3]

</details>

#### Q4. node 하나인 tree의 height와 root 성질은?

<details><summary>답 보기</summary>

edge 수 기준이면 root height는 0이고, 그 node는 root이면서 leaf다. 빈 tree height는 관례가 다르므로 별도로 선언한다. [출처: 01-21-root.md §14 Q4]

</details>

#### Q5. forest를 하나의 tree처럼 순회하려면?

<details><summary>답 보기</summary>

각 root에서 순회를 시작하거나, 실제 데이터에 없는 가상 super-root를 두고 각 tree root를 child로 연결하는 모델을 사용한다. [출처: 01-21-root.md §14 Q5]

</details>

F1. root는 트리의 시작점이 되는 노드이며 일반적으로 부모가 없는 최상위 노드다. [출처: NIST DADS Root (원본 01-21-root.md §0·§1 재인용)]

F3. 같은 무방향 tree에서 root 선택을 바꾸면 edge 집합은 같아도 parent, child, depth, subtree, traversal 시작점이 달라진다. [출처: 01-21-root.md §4]

## 모름 (U)

U1. 빈 tree의 height 규약(-1 또는 0 등)은 자료마다 달라 원본이 단일 값을 정하지 않았고, 이 문서도 정하지 않았다. [출처: 01-21-root.md §8]
U2. forest를 super-root로 묶는 모델의 구체 구현 비용은 원본에 근거가 없어 다루지 않았다.

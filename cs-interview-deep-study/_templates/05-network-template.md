# 05. 네트워크 오버레이 템플릿

> 먼저 `00-core-concept-template.md`를 적용하고, 네트워크 주제에는 아래 질문을 추가한다.

## 핵심 축

```text
어느 계층의 개념인가?
통신 주체는 누구인가? client, server, host, router, DNS resolver 중 무엇인가?
메시지 단위는 무엇인가? frame, packet, segment, request, response 중 무엇인가?
상태를 유지하는가, stateless인가?
```

## 메시지/상태 흐름

```text
요청 또는 패킷의 필드는 무엇인가?
연결 수립 단계가 있는가?
정상 송수신 순서는 어떻게 되는가?
손실, 지연, 중복, 순서 뒤바뀜이 있으면 어떻게 되는가?
타임아웃과 재시도는 누가 판단하는가?
```

## 실무 질문

```text
브라우저/클라이언트에서 서버까지 어떤 순서로 지나가는가?
로드밸런서, 프록시, TLS 종료 지점이 있는가?
keep-alive, connection pool, timeout 설정이 영향을 주는가?
패킷 캡처나 로그로 무엇을 확인할 수 있는가?
```

## 꼬리질문 축

```text
TCP와 UDP 중 왜 이것을 쓰는가?
HTTP는 TCP 위에서 어떤 문제를 푸는가?
TLS는 어느 구간을 보호하는가?
DNS가 느리거나 실패하면 전체 요청은 어떻게 되는가?
L4와 L7 로드밸런싱의 판단 기준은 무엇인가?
```

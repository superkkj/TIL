# 06. 데이터베이스 오버레이 템플릿

> 먼저 `00-core-concept-template.md`를 적용하고, DB 주제에는 아래 질문을 추가한다.

## 핵심 축

```text
데이터 단위는 무엇인가? row, page, index entry, transaction 중 무엇인가?
문제는 읽기 성능, 쓰기 성능, 정합성, 격리, 복구 중 무엇인가?
쿼리 경로에서 옵티마이저, 인덱스, 락, 로그 중 무엇이 개입하는가?
```

## 쿼리/저장 경로

```text
SQL 또는 요청은 어떤 단계로 처리되는가?
파싱, 최적화, 실행, fetch 중 어디가 핵심인가?
인덱스를 타는가, full scan인가?
메모리 버퍼와 디스크 I/O가 어떻게 연결되는가?
EXPLAIN으로 무엇을 확인할 수 있는가?
```

## 트랜잭션/락 질문

```text
ACID 중 어떤 성질과 연결되는가?
격리수준에 따라 어떤 현상이 허용되는가?
락은 row, gap, table, metadata 중 어디에 걸리는가?
deadlock은 어떻게 생기고 어떻게 감지/해소되는가?
undo/redo log가 필요한가?
```

## 인덱스 질문

```text
인덱스 key는 무엇인가?
선택도는 높은가?
범위 조건, 정렬, 조인에 도움이 되는가?
복합 인덱스라면 컬럼 순서가 왜 중요한가?
쓰기 비용과 저장 공간 비용은 무엇인가?
```

## 꼬리질문 축

```text
왜 Hash 인덱스가 아니라 B+Tree인가?
N+1은 쿼리 경로에서 어디가 문제인가?
격리수준을 높이면 무엇을 얻고 무엇을 잃는가?
락을 줄이면 어떤 정합성 문제가 생기는가?
캐시를 붙이면 DB 정합성은 어떻게 되는가?
```

# 04. 운영체제 오버레이 템플릿

> 먼저 `00-core-concept-template.md`를 적용하고, OS 주제에는 아래 질문을 추가한다.

## 핵심 축

```text
관리 대상 자원은 무엇인가? CPU, memory, file, device, process 중 무엇인가?
사용자 모드와 커널 모드 중 어디서 일어나는가?
프로세스 단위인가, 스레드 단위인가?
주소 공간, 실행 흐름, 자원 소유권 중 무엇이 핵심인가?
```

## 상태와 전이

```text
가능한 상태는 무엇인가?
상태 전이는 어떤 이벤트로 발생하는가?
커널 자료구조에는 어떤 정보가 저장되는가?
문맥 전환이 필요한가?
시스템 콜 경계가 있는가?
```

## 메모리 질문

```text
가상 주소와 물리 주소가 구분되는가?
page, frame, segment 중 무엇이 등장하는가?
TLB나 page table이 관련되는가?
보호와 격리는 어떻게 보장되는가?
page fault가 나면 어떤 순서로 처리되는가?
```

## 스케줄링/자원 질문

```text
선점형인가 비선점형인가?
ready, running, blocked 상태가 있는가?
공정성, 처리량, 응답시간 중 무엇을 최적화하는가?
I/O 대기와 CPU 사용은 어떻게 구분되는가?
```

## 꼬리질문 축

```text
프로세스와 스레드의 자원 공유 차이는?
context switch 비용은 어디서 발생하는가?
kernel mode 전환은 왜 비싼가?
stack은 왜 스레드별로 필요한가?
heap은 왜 프로세스 안에서 공유되는가?
```
